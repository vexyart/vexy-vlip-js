## REFERENCE

For a normal HTTPS video URL, the main “tool” is just the browser’s built-in media stack:

```js
const url = "https://example.com/video.mp4";

const video = document.createElement("video");
video.crossOrigin = "anonymous";     // needed if you want canvas pixels
video.muted = true;
video.preload = "metadata";
video.src = url;

await new Promise((resolve, reject) => {
  video.onloadedmetadata = resolve;
  video.onerror = reject;
});

// Seek near the start.
// `0` sometimes returns black/slate depending on encoding, so tiny offset is often safer.
video.currentTime = Math.min(0.05, video.duration || 0.05);

await new Promise((resolve, reject) => {
  video.onseeked = resolve;
  video.onerror = reject;
});

const canvas = document.querySelector("canvas");
canvas.width = video.videoWidth;
canvas.height = video.videoHeight;

const ctx = canvas.getContext("2d");
ctx.drawImage(video, 0, 0, canvas.width, canvas.height);
```

The browser-native options, from simplest to fanciest:

### 1. `<video>` + `<canvas>`

Best default. Works with MP4/WebM/OGV/etc. as supported by the browser.

```js
ctx.drawImage(video, 0, 0);
```

Pros: simple, fast, no dependency.
Cons: needs CORS headers if you want to read/export the canvas. Without CORS, drawing may work visually, but the canvas becomes **tainted**, so `toBlob()`, `toDataURL()`, or `getImageData()` will fail.

The remote server should send something like:

```http
Access-Control-Allow-Origin: *
```

or your exact origin.

### 2. `createImageBitmap(video)`

Can be cleaner if you want an intermediate bitmap:

```js
const bitmap = await createImageBitmap(video);
ctx.drawImage(bitmap, 0, 0);
bitmap.close();
```

Still depends on loading/seeking the video first. Same CORS rules apply.

### 3. `requestVideoFrameCallback()`

Useful when you want to wait for the browser to actually decode a frame, not just fire `seeked`.

```js
await new Promise(resolve => {
  video.requestVideoFrameCallback(() => resolve());
});

ctx.drawImage(video, 0, 0);
```

Good for avoiding “why did I get a black frame?” goblins 🕳️.

You can combine it with seeking:

```js
video.currentTime = 0.05;
await new Promise(resolve => video.onseeked = resolve);

await new Promise(resolve => {
  video.requestVideoFrameCallback(resolve);
});

ctx.drawImage(video, 0, 0);
```

### 4. WebCodecs API

Use this if you need lower-level decoding, performance, batch frame extraction, or processing many clips.

Relevant APIs:

```js
VideoDecoder
EncodedVideoChunk
VideoFrame
```

But: WebCodecs does **not** magically parse MP4/WebM containers for you. You usually need a demuxer, for example:

```bash
mp4box.js
```

Typical stack:

```text
fetch(url)
→ demux with mp4box.js
→ decode with WebCodecs VideoDecoder
→ draw VideoFrame to canvas
```

This is powerful, but much more plumbing.

### 5. ffmpeg.wasm

For “I need this to work even with weird formats and exact-frame extraction”:

```text
@ffmpeg/ffmpeg
```

Pros: very flexible.
Cons: heavy, slow to load, CPU-hungry, not elegant for a quick thumbnail. It’s a forklift for a sandwich.

### 6. Media Source Extensions

Usually **not** needed just to get the first frame. It is useful if you are streaming fragmented media manually or working with HLS/DASH-like chunks.

### Most practical helper function

```js
async function videoFirstFrameToCanvas(url, canvas, time = 0.05) {
  const video = document.createElement("video");

  video.crossOrigin = "anonymous";
  video.muted = true;
  video.playsInline = true;
  video.preload = "auto";
  video.src = url;

  await new Promise((resolve, reject) => {
    video.onloadedmetadata = resolve;
    video.onerror = () => reject(new Error("Could not load video metadata"));
  });

  video.currentTime = Math.min(time, Math.max(0, video.duration - 0.01));

  await new Promise((resolve, reject) => {
    video.onseeked = resolve;
    video.onerror = () => reject(new Error("Could not seek video"));
  });

  if ("requestVideoFrameCallback" in HTMLVideoElement.prototype) {
    await new Promise(resolve => {
      video.requestVideoFrameCallback(() => resolve());
    });
  }

  canvas.width = video.videoWidth;
  canvas.height = video.videoHeight;

  const ctx = canvas.getContext("2d");
  ctx.drawImage(video, 0, 0, canvas.width, canvas.height);

  return canvas;
}
```

Usage:

```js
const canvas = document.querySelector("canvas");

await videoFirstFrameToCanvas(
  "https://example.com/clip.mp4",
  canvas
);
```

### The practical recommendation

Use **`<video>` + `canvas.drawImage()` + `requestVideoFrameCallback()`**.

Only reach for **WebCodecs** if you need precise decoding/control at scale. Only reach for **ffmpeg.wasm** if browser media decoding is not enough or you need format necromancy.
