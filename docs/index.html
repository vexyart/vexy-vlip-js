Your video already knows what to say.

Vexy Vlip treats a WebVTT track not as subtitles but as on-screen cards — styled, positioned, and timed. Play straight through, or click to step from one explanation to the next.

In stop-motion mode, click the video (or press Space) to advance to the next card. This is the real Vexy Playlines editor, captured and narrated with one .vtt file.

What it does

Cards, not captions

WebVTT cues render as real HTML panels — solid or translucent backdrops, custom fonts, colours, radius, borders, shadows. Crisp, accessible, responsive.

Placed where you mean

Use native WebVTT position/line/size hints, or put JSON in the cue payload for exact x/y/w and anchor control.

Stop-motion tutorials

Play to the next cue, pause, show the card; click to continue. Perfect for explainers and product walk-throughs.

Optional eased travel

Glide between stops with an ease-in-out curve — no MediaBunny, no WebCodecs, just native seeking.

Three builds, one core

Import the ESM module, drop in the <vexy-vlip> element, or load one global script. Same engine underneath.

Themeable

Every knob is a CSS custom property (--vv-card-bg, --vv-accent…). Style it to match your site.

Three ways to use it

ESM

JavaScript module

import { VexyVlip } from "vexy-vlip-js" and instantiate against any container.

 HTML

Web Component

Just write <vexy-vlip src track mode>. It registers itself.

 SCRIPT

Classic global

One <script> tag exposes window.VexyVlip for no-build pages.
